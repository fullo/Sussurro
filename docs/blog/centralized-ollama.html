Setup

Run Ollama on a separate machine

Sussurro's AI cleanup runs on Ollama over HTTP — and that server doesn't have to be the same computer you dictate on. Run it once on a capable box and let every laptop share it.

6 min read · Setup guide

Every Sussurro install can run its own local Ollama, but you don't have to. Because the cleanup step is just an HTTP call, you can host Ollama on one machine — a desktop with a GPU, a home server, a spare Mac — and point all your other devices at it. One place to install the model, one place to update it, one place doing the heavy lifting.

Why centralize

  • Install the model once. Pull llama3.2:3b on the server and every device uses it — no per-laptop downloads.
  • Put the compute where the hardware is. A desktop GPU cleans faster than a thin laptop; the laptop just records and pastes.
  • Update in one place. Swap the model or upgrade Ollama on the server; clients need no changes.

Step 1 — Make Ollama listen on the network

By default Ollama binds to localhost, so it only accepts calls from the same machine. Tell it to listen on all interfaces by setting OLLAMA_HOST:

# quick test (foreground)
OLLAMA_HOST=0.0.0.0:11434 ollama serve

To make it permanent, set the environment variable for the Ollama service:

  • Linux (systemd): systemctl edit ollama and add Environment="OLLAMA_HOST=0.0.0.0:11434", then systemctl restart ollama.
  • macOS: launchctl setenv OLLAMA_HOST "0.0.0.0:11434" and restart the Ollama app.
  • Windows: set a user environment variable OLLAMA_HOST = 0.0.0.0:11434 and restart Ollama.

Step 2 — Pull the model on the server

ollama pull llama3.2:3b

Not sure which model? See Choosing the right Ollama model.

Step 3 — Point Sussurro at it

  1. Open Settings → Cleanup.
  2. Set the Server field to http://SERVER-IP:11434 (e.g. http://192.168.1.20:11434).
  3. The model dropdown loads live from the server's /api/tags — pick your model.

That's it. Dictate as usual; the transcript is sent to the server for cleanup and the cleaned text comes back to be pasted locally. If the server is unreachable, Sussurro falls back to the raw transcript — you're never blocked.

Security first

Ollama's API has no authentication. Anyone who can reach SERVER-IP:11434 can use your model. Keep it on a trusted LAN, or better, a private network like Tailscale/WireGuard. Never forward port 11434 to the public internet.

What about latency?

On a LAN, the round-trip to a small model is negligible — a 2–3B model on a decent machine cleans a sentence in well under a second. With streaming typing and cleanup enabled, Sussurro makes one call per finished sentence; over Wi-Fi or Ethernet this stays smooth. Over a slower link (a remote server across the internet via VPN), prefer batch cleanup (paste on release) rather than per-sentence streaming.

Privacy note

Centralizing Ollama keeps everything on your hardware — the audio is still transcribed locally on each device, and only the text transcript travels to your own server for cleanup. Nothing leaves the machines you control.
