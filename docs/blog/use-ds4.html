Setup

Use DS4 — or any OpenAI-compatible server — with Sussurro

Since 0.6.0, Sussurro's cleanup can talk the OpenAI-compatible API, not just Ollama's native one. That opens the door to any local runtime that speaks /v1/chat/completions — including antirez's DS4.

5 min read · Setup guide

Sussurro cleans up your dictation with a local LLM. Until 0.6.0 that model had to be served by Ollama, through Ollama's own API. Now there's a second option: the OpenAI-compatible API — the de-facto standard /v1/chat/completions that most local runtimes expose. Point Sussurro at any of them and it drives cleanup and command mode exactly the same way.

What is DS4?

DS4 (DwarfStar 4) is Salvatore Sanfilippo's (antirez) local inference engine, written in C to run DeepSeek V4 — a frontier-class model — on your own machine, with an OpenAI/Anthropic-compatible server built in. It's a remarkable piece of engineering.

Reality check

DS4 runs a single 284B-parameter model and wants 96–128 GB of RAM (an M-series Mac with 128 GB, or a big CUDA/ROCm box). It is not a laptop model, and it's massive overkill for the small "remove the ums, fix the punctuation" job cleanup actually needs. The point of 0.6.0 isn't "use DS4 for everything" — it's that Sussurro no longer cares which server runs the model, so you can point it at whatever you already run.

Point Sussurro at it

  1. Start your OpenAI-compatible server (DS4 in server mode, or another — see below).
  2. In Sussurro, open Settings → Cleanup and set Cleanup backend to OpenAI-compatible.
  3. Set Server to the server's base URL — e.g. http://localhost:8080. A trailing /v1 is optional; Sussurro normalizes it.
  4. Pick the model (the list loads from the server's /v1/models; type it by hand if the server doesn't expose one).
  5. If your server requires a token, fill in API key — most local servers ignore it, so leave it empty otherwise.

That's it. Dictate as usual; the transcript is cleaned by whatever server you pointed at, with the same graceful fallback to the raw transcript if it's unreachable.

Not just DS4

The same OpenAI-compatible setting works with any server that speaks /v1:

  • llama.cpp-server — the reference lightweight server; run any GGUF model.
  • LM Studio — its local server exposes an OpenAI-compatible endpoint.
  • vLLM and other inference servers.
  • Ollama itself — it also serves /v1, so the easiest way to try the new backend is to keep Server at http://localhost:11434 and just switch the backend to OpenAI-compatible. It should clean identically to native mode.
Requires 0.6.0+

The OpenAI-compatible backend arrived in Sussurro 0.6.0. Ollama's native API remains the default — nothing changes unless you switch it. Grab the latest from the download page or the in-app updater.

Which model should you actually pick for cleanup? A small instruct model is plenty — see choosing the right model.
