Concept

Clipboard-paste injection, explained

Sussurro doesn't integrate with any app's API — it puts text on the clipboard, pastes it, and restores what was there before.

5 min read · Concept

Sussurro has to type into whatever application happens to have focus — a browser's compose box, a terminal, a native editor, a chat client, a form field buried three dialogs deep. There's no universal "insert text here" API that works across all of them. So Sussurro doesn't try to integrate with each app. Instead it uses the one interface every app already understands: the clipboard.

The sequence

When it's time to deliver text — a finished dictation, a streamed sentence, an edited command-mode selection — Sussurro runs the same four-step sequence, implemented in inject.rs:

  1. Read and hold whatever is currently on the clipboard.
  2. Write the new text to the clipboard.
  3. Synthesize a paste keystroke (Ctrl+V, or Cmd+V on macOS) into the focused window.
  4. Restore the original clipboard contents.

A short pause is built into the middle of that sequence: after writing the clipboard, Sussurro waits briefly for the OS to propagate the change before sending the paste keystroke, and after pasting it waits again before restoring — giving the target app time to actually read the clipboard before its old contents come back.

Why paste instead of typing

The alternative — synthesizing a keystroke for every character — works almost everywhere too, but it's slower for longer passages and more likely to trip up apps with custom text-input handling (rich editors, some Electron apps, certain terminal emulators) that don't process synthetic per-key events cleanly. Pasting a whole block of text in one keystroke sidesteps that class of problem, which is why Sussurro treats it as the default delivery path rather than a fallback. The trade-off is the one honest cost of this approach: your clipboard is touched, even if only for a couple hundred milliseconds.

YOUR CLIPBOARD IS SAFE

The swap is save → write → paste → restore, always in that order. If you had something copied before triggering dictation, it's back on your clipboard by the time the paste completes — you don't need to re-copy it.

Where it shows up in the pipeline

This isn't only a one-shot operation at the end of a dictation. Streaming typing calls the same inject_text routine once per delta or per cleaned sentence as you speak, so a single dictation can trigger several quick clipboard swaps in a row rather than one big one at the end. Command mode uses the same machinery in reverse: it copies your current selection (via a synthesized Ctrl+C) before rewriting it, then pastes the edited result back over the selection.

Edge cases

  • Apps that block or intercept paste. A small number of hardened or sandboxed apps disable programmatic paste, or don't expose a focused text field the way the OS expects. When that happens the keystroke lands but nothing appears — there's no per-app fallback beyond the clipboard-and-paste flow itself.
  • No clipboard manager conflicts. Because Sussurro restores your previous clipboard content immediately after pasting, clipboard-history tools that watch for changes may briefly see Sussurro's text appear and disappear. This is expected, not a bug.
  • Nothing selected in command mode. If you invoke command mode with no text selected, Sussurro finds nothing to copy and puts your original clipboard back untouched rather than pasting over random content.

The Linux Wayland wrinkle

On Wayland, the clipboard-paste flow is treated as a fallback rather than the primary path. Wayland compositors like KWin and Mutter deliberately restrict low-level input injection, so Sussurro first tries the XDG RemoteDesktop portal to type text directly — no clipboard involved at all. Only when that portal (or the native-tool ladder behind it) isn't available does Sussurro fall back to the same clipboard-and-paste sequence used on Windows and macOS, using wl-clipboard for the clipboard read/write. The full story, including the one-time consent dialog and its fallbacks, is in Wayland injection, explained.
