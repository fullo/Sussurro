Per-app tone styles

The same sentence should sound different in a Slack thread and a client email. Sussurro notices which app you're dictating into and adapts the cleanup to match.

5 min read · Feature

You don't write the same way everywhere. Chat is loose and quick; email is measured; a commit message is terse. Per-app tone styles let you encode that once and forget it — Sussurro detects the focused app the moment you release the trigger and folds a tone instruction into the cleanup prompt.

How rules work

A rule maps an app to a tone. For example:

  • slack → “casual, emojis welcome, keep it short”
  • mail → “polite and professional, full sentences”
  • code → “terse, no fluff”

When you dictate into a matching app, that instruction shapes how the LLM cleans your words. Everywhere else, your normal cleanup applies.

Force a language per app

Each rule can also set its own output language, overriding the global Translate to. Dictate in Italian and have Slack come out in English while WhatsApp stays Italian — no toggling settings between messages.

Detected at release

The app in focus is read at the instant you finish speaking — so if you trigger dictation, switch windows, then release, the tone matches where the text will actually land.

Setting them up

  1. Go to the Personalization section.
  2. Add a rule: the app identifier plus a short tone description (and optionally a forced language).
  3. Dictate into that app and watch the cleanup adapt.

Tone rules travel with your portable config, so your whole set moves to a new machine in one JSON file.
