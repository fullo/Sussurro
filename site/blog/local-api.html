The local API

Sussurro can expose a small HTTP endpoint on loopback so your own scripts can clean text, transcribe audio, and search history — using your live settings.

5 min read · Feature

The pipeline that powers dictation is useful outside dictation too. The local API lets any script on your machine tap into it — the same cleanup model, the same STT engine, the same history — over plain HTTP.

Turn it on

It ships off by default. Enable it in Behavior → Advanced; it binds to loopback only and applies at restart.

The endpoints

# clean up / translate a text with your current settings
curl -X POST --data "um so this is uh a test" http://127.0.0.1:4525/clean

# transcribe an audio file (wav/mp3/m4a/flac/ogg)
curl -X POST --data-binary @meeting.mp3 "http://127.0.0.1:4525/transcribe?ext=mp3"

# search your dictation history
curl "http://127.0.0.1:4525/history?n=10&q=sussurro"

What you can build

  • A shell alias that cleans clipboard text on demand.
  • A batch job that transcribes a folder of voice memos overnight.
  • An editor plugin that pipes selected text through your cleanup prompt.
  • A script that pulls the last n dictations into your notes.
Security

Loopback-only means it isn't exposed to the network — but any process on your machine can call it. That's exactly why it ships disabled; turn it on only when you want it, on a machine you trust.

Because everything runs locally, the API is as private as the app: your text and audio go to your own model and your own disk, never to a server.
