Shortcuts & trigger modes: push-to-talk vs toggle

Push-to-talk or toggle, one shortcut for dictation and a separate one for command mode — how the click-to-record widget and the header Dictate button fit together.

4 min read · Setup guide

Sussurro is triggered two ways: a system-wide keyboard shortcut that works from any app, or the Dictate button in the header when the window is focused. Both share the same underlying setting for one important choice — do you hold to record, or tap to start and stop? There's also a second, separate shortcut for command mode, which edits text instead of writing it.

Setting the dictation shortcut

In Settings → Dictation, the Shortcut field is a live recorder, not a text box. Click it, then press the key combination you want — Sussurro captures it directly from the keydown event and shows Esc as the way out if you change your mind. The default is Ctrl+Shift+Space (⌘+Shift+Space on macOS).

  1. Open Settings → Dictation.
  2. Click the Shortcut field — it switches to "Press keys…".
  3. Press the combination you want. A bare letter or number is rejected (it would hijack normal typing) unless it's an F key; everything else needs at least one modifier.
  4. The new combination is registered immediately.

Push-to-talk vs toggle

A single switch — Push-to-talk, also in Settings → Dictation — decides how both the shortcut and the Dictate button behave:

  • On (default): recording lasts exactly as long as you hold the shortcut or the button down, like a walkie-talkie. Release, and Sussurro stops and transcribes.
  • Off (toggle): one press starts recording, a second press stops it. Useful if holding a key for a long dictation is uncomfortable, or if your hardware doesn't reliably report key-up events.

This one setting covers both triggers — there's no separate push-to-talk flag for the keyboard versus the button.

Header pill = Dictate button

The status pill next to the Sussurro wordmark is a real button, not just a status indicator. Hold it (push-to-talk) or click it (toggle) to dictate with the mouse instead of the keyboard — handy when the app is already focused. Dragging off the button while holding it stops the recording, the same as releasing a held key.

The separate command-mode shortcut

Command mode has its own field, Command shortcut, defaulting to Ctrl+Alt+Space. It's registered independently from the dictation shortcut and configured with the same click-to-record widget. Select text anywhere, trigger the command shortcut, and speak an instruction — Sussurro sends it to the local LLM and replaces the selection with the result, instead of pasting new text.

The command shortcut follows the same push-to-talk/toggle setting as dictation, since it's one switch for both triggers. With push-to-talk on (the default), you hold the command shortcut while speaking the instruction and release when you're done.

For what to say once command mode is active, see command mode.

Picking combinations that don't collide

Both shortcuts are registered as global OS-level hotkeys, so they need to be free of conflicts with your window manager, other apps, and each other. If a combination fails to register — a policy restriction on some Wayland compositors, for instance, or another app already holding it — dictation itself is never blocked: a failed shortcut registration falls back gracefully, and the Dictate button keeps working regardless.

Tip

If you dictate mostly in short bursts, push-to-talk keeps you from accidentally leaving the mic open. If you dictate long paragraphs, toggle mode saves your hand from holding a chord the whole time.
