Where your data lives (and moving the models folder)

A map of every local file Sussurro writes — models, history, settings, dictionary — and how to move the models folder to a bigger disk.

5 min read · Setup guide

Sussurro is a local-first dictation app: no account, no cloud sync, no telemetry server holding your transcripts. Everything it produces — models, history, settings, dictionary — is a plain file on your own disk, in a location the operating system already gives every desktop app. This is a map of what lives where, and how to move the one folder that can get large: the models directory.

What gets stored, and as what

  • Settings — hotkeys, cleanup level, engine choice, per-app styles, and every other toggle in the Settings panel are saved as settings.json in the app's config directory. It's written automatically whenever you change a setting.
  • Dictation history — every completed dictation (raw transcript and cleaned text, with a timestamp) is appended as one JSON object per line to a history.jsonl file in the app's data directory. Being JSONL rather than a database means you can grep it, tail it, or parse it with any scripting language.
  • Usage statistics — total/today/last-7-days counts live in a small stats.json alongside the history file, tracked separately so clearing history never resets them.
  • STT models — the Whisper .bin file or the Parakeet model bundle live in a models subfolder, which is the one directory large enough to matter (multi-GB depending on model size).
  • Dictionary, snippets, app styles — these live inside settings.json too, but they're also the pieces you can export separately (see below) since they're the part of your setup worth carrying to another machine.

Sussurro asks the OS (via Tauri) for its standard per-user config and data directories rather than hardcoding a path, so the exact location follows your platform's convention — the point is that it's always a normal folder under your own user account, never anything remote.

NOTHING LEAVES YOUR MACHINE

Transcription runs through whisper.cpp or Parakeet locally, cleanup goes to your own local Ollama instance (or a LAN one, see centralized Ollama), and injection pastes straight into the focused app. None of the files described here are ever uploaded anywhere by Sussurro itself.

Moving the models folder

Whisper and Parakeet models are the only thing here that can genuinely strain a small system drive. Settings has a Models folder field, under the advanced group, that overrides where Sussurro looks for and downloads models:

  1. Open Settings and find Models folder (it shows empty = app data default underneath).
  2. Type the path to a folder on a roomier disk, for example a dedicated models directory on a secondary drive.
  3. If you already have models downloaded under the old (default) location, move the files there yourself first — Sussurro does not migrate them automatically, it only starts looking in the new folder from then on.
  4. Leave the field empty to go back to the default app-data location at any time.
MANUAL MOVE

Changing the setting does not copy or delete anything — it only changes where Sussurro looks next. If the model file isn't present at the new path, Sussurro will treat it as missing and offer to download it again.

Portable config: taking your setup with you

Your personal dictionary, voice snippets, and per-app tone styles are the parts of a setup that are genuinely worth reusing on a second machine — unlike the hotkey or the models folder path, which are tied to that machine. Settings has an export/import pair for exactly this: exporting writes those three things to a JSON file you choose, and importing merges them into the current settings without creating duplicates. Because it's just a file, you can sync it yourself with Git, Syncthing, or a shared drive — no account required, same as everything else Sussurro touches.

Cleaning up

History has its own retention setting — auto-delete entries older than N days, or keep everything forever with the default of 0 — plus a manual clear and Markdown/JSON export from the History panel. Because history and settings are separate files, clearing your dictation history never touches your dictionary, snippets, or usage totals.

The short version: nothing about where your data lives is a mystery or a black box. It's a handful of ordinary files under your own user account, one setting to relocate the big one, and one export button if you ever want to carry the rest somewhere else.
